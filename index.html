Fundamentos del Desarrollo Móvil

01¿Qué es un sistema operativo móvil?+

Es el software base que gestiona el hardware y los recursos de un dispositivo móvil. Coordina funciones como pantalla táctil, conectividad, sensores y llamadas, y permite ejecutar aplicaciones.

AndroidiOSHarmonyOSWindows Phone
02Describe las cinco etapas del desarrollo móvil+
1
Análisis — Se identifican las necesidades del cliente y los requisitos funcionales y no funcionales.
2
Diseño — Se definen la arquitectura, la interfaz gráfica y la UX (wireframes, prototipos).
3
Desarrollo — Se escribe el código fuente usando los frameworks y lenguajes elegidos.
4
Pruebas — Se verifican funcionamiento, rendimiento y seguridad para detectar y corregir errores.

PARCIAL 03

Depuración, Versiones y Publicación

26Verdadero (1) o Falso (0) — Errores, versiones y documentación+
1Los errores de sintaxis se presentan en líneas que no cumplen las reglas de escritura.
2Un sistema controlador de versiones permite conocer el estado de las variables durante la ejecución.
3La documentación solo registra errores durante el desarrollo y pruebas.
4El nombre "viajes delUsuaio.js" es adecuado para un componente de lista desplegable.
5Los comentarios multilínea inician con dos diagonales (//).
6Mantener actualizadas las dependencias previene errores de compatibilidad.
27Procedimiento para usar un sistema controlador de versiones (6 etapas)+
1
Inicializar el repositorio — Se crea el repositorio local con git init.
2
Conectar al remoto — Se vincula con un repositorio en la nube (GitHub, GitLab).
3
Realizar cambios — El desarrollador edita, agrega o elimina archivos del proyecto.
4
Registrar (commit) — Se guarda el historial con un mensaje descriptivo.
5
Subir al remoto (push) — Se sincroniza el repositorio local con el remoto.
6
Revertir o fusionar — Se recupera una versión anterior o se combinan ramas en caso de conflictos.

30¿En qué consiste el proceso de mantenimiento de una app?+
Consiste en dar seguimiento continuo a la app tras su publicación. Incluye corregir errores, actualizar bibliotecas, mejorar el rendimiento, adaptar la app a nuevas versiones del SO y agregar funcionalidades según la retroalimentación de los usuarios.
31¿Cuáles son las tres fases del proceso ASO?+
1
Investigación
Análisis de palabras clave, competencia y comportamiento del público objetivo.
2
Optimización
Aplicar resultados al título, descripción, íconos y capturas de la ficha técnica.
3
Seguimiento
Monitorear métricas de descargas, valoraciones y conversiones para mejorar continuamente.
328 pasos para distribuir una app en una tienda de aplicaciones+
1
Crear una cuenta de desarrollador en la tienda (Google Play / App Store).
2
Compilar la aplicación generando el archivo instalable (APK / IPA).
3
Crear la ficha técnica (nombre, descripción, categoría, capturas).
4
Configurar precios y países de distribución.
5
Definir la clasificación de contenido (edad y permisos).
6
Subir el archivo compilado de la aplicación.
7
Enviar la app para revisión y aprobación por la tienda.
8
Publicar y dar seguimiento a la retroalimentación de los usuarios.
